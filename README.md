# JAPAN NOW

「今、日本で何が起きているか」をひと目で把握するための、個人プロジェクトのダッシュボード。

姉妹プロジェクトの [ORACLE (World Risk Intelligence)](../oracle-main) とはエンジニアリング上の作法(キャッシュ・degraded表示・テストの習慣など)を共有しているが、コンセプトは意図的に違う。

## これは何であって、何でないか

- **スコアやランキングは一切算出しない。** ORACLEの「Global Risk Index」のような数値化・重み付けの仕組みはここにはない。ニュース・気象警報・地震情報は「足し合わせるリスクポイント」ではなく、ただの分類された事実として扱う。
- **予測システムではない。** 表示しているのは自動取得した公開情報であり、編集・査読は行っていない。
- **全国対応。** ニュース・気象警報・地震情報は全国をカバーしている。

## データソース

| セクション | ソース | 備考 |
|---|---|---|
| ニュース(全国) | NHK NEWS WEB RSS | 非公式URL、将来変わる可能性あり |
| ニュース(国政) | Google News (クエリ検索) | 国会・内閣・首相など政治/経済/災害キーワードで絞り込み |
| 気象警報・注意報 | 気象庁 warning API | 非公式(気象庁が公開しているJSON、正式サポートAPIではない) |
| 地震・火山情報 | 気象庁防災情報XML(eqvol.xml) | 公式のPULL型フィード |

**あえて使っていないもの:**
- Yahoo!ニュースRSS — 利用規約でサイト/アプリへの組み込みが明確に禁止されている
- NHKの「地域ニュース一覧」ページ — 閲覧に利用意向の確認(同意ステップ)が必要な作りになっており、機械的な取得は規約的にグレーと判断
- 運行情報(ODPT) — 検討したが、無料枠で現実的にカバーできるのが東京メトロ・都営地下鉄程度で「全国の鉄道」には程遠く、いったん見送り。鉄道会社ごとに参加状況が違うため、気象庁のように1つのAPIで全国を賄う構造にならない。
- 地方ニュース専用セクション — 47都道府県名のOR検索を1本の独立セクションとして試したが、Google Newsの関連度ランキングが「複数県名に同時マッチする記事」(天気関連の広域記事など)を優先するため、結局は一部地域(北海道・東北)に偏る結果になった。1県あたりの件数上限で偏りは抑えられても、「47都道府県すべてに公平にチャンスがある」状態にはならず、本当に全国対応にするには都道府県ごとに個別クエリを47本投げる必要があり、運行情報の時と同じ理由(コストと構造上の限界)で見送った。

## 設計上の原則

1. **「取得失敗」と「本当に何もない」を混同しない。** 気象庁への通信が全部失敗しても、「警報はありません」と表示してはいけない。各セクションは`○○Ok`という成否フラグを持ち、失敗時は明示的にエラー表示になる。
2. **架空のデータソースを作らない。** 実際に叩いていないAPIを「対応済み」であるかのように見せない。
3. **キーワードマッチの限界を隠さない。** ニュースの絞り込みはキーワードベースであり、意味理解はしていない。風刺記事の誤検出、見出しの局名クレジット漏れなど、実際にバグとして踏んだものはコード内のコメントに経緯を残している。

## 既知の限界

- ニュースのキーワード絞り込みは完全ではない(誤検出・見逃しどちらもあり得る)。特に「日本に関する話かどうか」は判定しておらず、「選挙」「事故」等の一般的なキーワードは海外の同種のニュースにもヒットする。ロシア・ウクライナ・イスラエル・ガザ・シリア・レバノン・ライアンエアー・イラン・英国・アルゼンチンなど特定の海外シグナル語を含み、かつ日本関連語(日本・日米・日中・来日・訪日など)を含まない記事は除外しているが、これは限定的なリスト頼みの対症療法であり、全ての海外ニュースを排除できるわけではない
  - **意図的にブロックリストへ入れない国**: 中国・米国・韓国など、日本と常に密接に絡む主要国はこのリストに追加しない方針。「米、中国の選挙介入能力巡る機密情報公開を検討か」のような、たまたま日本と無関係な米中間の記事が「選挙」のような緩いキーワードで紛れ込むことはあるが、これらの国名をブロックリストに入れると、日中関係・日米安保・対中輸出規制のような、本当に載せるべき記事まで大量に誤って消してしまうリスクの方が大きいと判断した。ロシア・イラン・英国のような、このサイトの文脈でほぼ日本と絡まない国とはリスクの性質が違う
- NHK RSSのURLは非公式で、予告なく変わる可能性がある
- 気象庁への同時アクセス数(都道府県別の警報を全件取得するため)は、アクセスが増えた場合に見直しが必要かもしれない
- 運行情報(電車遅延等)は非対応。ODPTを検討したが、鉄道会社ごとに参加状況がバラバラで、気象庁のように1つのAPIで全国を賄えないため見送った
